## Настройки программы (Perl)
- `use strict;`  
  Включает строгий режим, требующий явного объявления переменных (`my`, `our`) и предотвращающий небезопасные конструкции. Повышает безопасность и читаемость кода.

- `use warnings;`  
  Включает вывод предупреждений о потенциальных ошибках (например, неинициализированных переменных), помогая выявлять проблемы на этапе разработки.

- `use utf8;`  
  Указывает, что исходный код программы использует кодировку UTF-8, что позволяет корректно обрабатывать символы, включая кириллицу.

- `binmode STDOUT, ':encoding(UTF-8)';`  
  Устанавливает кодировку UTF-8 для вывода в консоль, обеспечивая правильное отображение русских и других не-ASCII символов.

- `binmode STDIN, ':encoding(UTF-8)';`  
  Устанавливает кодировку UTF-8 для ввода с клавиатуры, позволяя корректно считывать строки с русскими символами.

## Описание функций для работы с массивами

#### read_array
```sub read_array { ... }``` Объявление функции с именем read_array.


```my $prompt = shift;```

shift без аргументов внутри подпрограммы берёт первый аргумент из @_ (списка аргументов функции).
Результат сохраняется в локальную переменную $prompt.

```my @arr;``` Создаётся пустой массив @arr, в который будут собираться введённые строки.

```while (my $line = <STDIN>) { ... }``` Читает построчно из STDIN в переменную $line.

```chomp $line;``` Убирает завершающий символ новой строки (по умолчанию \n) из $line.

```last if $line eq '';``` Если после chomp переменная пуста (пользователь ввёл пустую строку и нажал Enter), выполняется last — выход из цикла.

```push @arr, $line;``` Добавляет (вставляет в конец) текущую строку в массив @arr.

```return @arr;``` Возвращает массив.

#### sort_list

```my @list = @_;``` Все переданные аргументы функции помещаются в локальный массив @list.

```return () unless @list;``` Если массив пустой (@list равен 0 элементов), сразу возвращается пустой список.

```my $all_int = 1;``` Флаг, указывающий, что пока мы предполагаем: «все элементы массива — целые числа».

```for (@list) { ... }``` Перебираем каждый элемент массива по очереди.

```defined $_ && /^-?\d+$/``` проверка, что элемент существует и соответствует регулярному выражению ^-?\d+$, которое означает:

опциональный минус (-?)
одно или несколько десятичных цифр (\d+)
никаких других символов.

То есть это строго целое число (например, 123, -45, но не 3.14 и не "abc").

Если проверка не прошла, ставим $all_int = 0.

```last unless $all_int;``` — как только найден хотя бы один «нецелый» элемент, цикл прерывается.

```return $all_int ? (sort { $a <=> $b } @list) : (sort @list);```

Если $all_int всё ещё равен 1 → сортируем числа числовым оператором <=>.
Если $all_int сброшен в 0 → сортируем строки обычным sort (лексикографически).

#### operations_sets

```my ($a1, $a2) = @_;``` Функция принимает ссылки на два массива.

```$a1``` → ссылка на первый массив
```$a2``` → ссылка на второй массив
```
my %seen1 = map { $_ => 1 } @$a1;
my %seen2 = map { $_ => 1 } @$a2;
```
Создаются два хеша (%seen1 и %seen2) для быстрого поиска уникальных элементов.

Ключи хеша — элементы массива, значения (1) служат просто как заглушка.

Использование хеша автоматически убирает дубликаты.

Объединение
```
my %union_h = (%seen1, %seen2);
my @union = sort_list(keys %union_h);
```

Создаём новый хеш, объединяя ключи двух хешей.

```keys %union_h ```→ уникальные элементы обоих массивов.
```sort_list``` сортирует элементы (числово или лексикографически).

Пересечение

```my @intersection = sort_list(grep { exists $seen2{$_} } keys %seen1);``` 
Берём ключи первого хеша.

grep — фильтрует список, оставляя только элементы, которые удовлетворяют условию.

```{ exists $seen2{$_} }``` — анонимная функция (блок), которая проверяет:

есть ли текущий элемент $_ как ключ в хеше %seen2.

exists возвращает true, если ключ существует, и false, если нет.

Результат — элементы, общие для обоих массивов.

Разность A - B

```my @diff_ab = sort_list(grep { not exists $seen2{$_} } keys %seen1);```

Берём ключи первого хеша, которых нет во втором.

Результат — элементы первого массива, отсутствующие во втором.

Разность B - A

```my @diff_ba = sort_list(grep { not exists $seen1{$_} } keys %seen2);```

Аналогично, но в обратном порядке.

Симметричная разность
```
my %sym_h = map { $_ => 1 } (@diff_ab, @diff_ba);
my @sym_diff = sort_list(keys %sym_h);
```
Объединяем разности двух массивов.
Преобразуем в хеш, чтобы убрать дубликаты.
Сортируем и получаем конечный массив.

Печать результатов

```print "Объединение: ", @union ? join(' ', @union) : "(пусто)", "\n";```

Проверяется, пустой ли массив.
Если нет — соединяем элементы через пробел и выводим.
Если пустой — выводим (пусто).
Аналогично для пересечения, разностей и симметричной разности.

#### swap_pairs

```my ($a1) = @_;``` Функция получает ссылку на массив.
```$a1``` — это ссылка на первый массив.

```my @swapped = @$a1;``` Создаём локальную копию массива, чтобы не изменять исходный массив.

```@$a1``` разыменовывает ссылку, превращая её в обычный массив.

Цикл ```for (my $i = 0; $i+1 < @swapped; $i += 2)``` Перебираем индексы массива с шагом 2.

Условие ```$i+1 < @swapped``` гарантирует, что всегда есть соседний элемент для обмена.

```($swapped[$i], $swapped[$i+1]) = ($swapped[$i+1], $swapped[$i]);```

Используем списковое присваивание для обмена двух элементов.

```print @swapped ? join(' ', @swapped) . "\n" : "(пустой массив)\n";``` 
Если массив не пустой — соединяем элементы через пробел и выводим.
Если пустой — выводим сообщение (пустой массив).

#### merge_alternate

```my ($a1, $a2) = @_;``` Функция получает ссылки на два массива.

```$a1 ```→ ссылка на первый массив
```$a2``` → ссылка на второй массив

```my $min = @$a1 < @$a2 ? @$a1 : @$a2;``` Определяем длину меньшего массива.

```@$a1 ```и ```@$a2``` разыменовывают ссылки в массивы.

```$min``` — максимальное количество элементов, которое можно безопасно чередовать.

```my @merged;``` Создаём новый массив для результата.

Цикл ```for my $i (0..$min-1)```

Проходим по индексам от 0 до $min-1.

```push @merged, $a1->[$i], $a2->[$i];```

```->[$i]``` — обращение к элементу массива по индексу через ссылку.

Добавляем сначала элемент из первого массива, затем из второго.

```print @merged ? join(' ', @merged) . "\n" : "(пусто)\n";```

Если массив не пустой — соединяем элементы через пробел и выводим.
Если пустой — выводим (пусто).

#### menu_arrays

```my ($a1_ref, $a2_ref) = @_;``` Функция получает ссылки на два массива.

```$a1_ref ```→ ссылка на первый массив
```$a2_ref ```→ ссылка на второй массив

```while (1) { ... }```

Бесконечный цикл, который обеспечивает повторное отображение меню после каждой операции.
Цикл завершится только при выборе пункта «Назад» (4).

Вывод меню:

```print "\nМеню работы с массивами:\n";
print "1. Операции множеств ...\n";
...
```

```chomp(my $c = <STDIN>);``` Читаем ввод пользователя с клавиатуры.

```chomp``` удаляет символ новой строки \n, чтобы можно было корректно сравнивать выбор.

Выбор действия через условный оператор if ... elsif ... else:

```if ($c eq '1')``` → вызывается функция operations_sets для операций над множествами.

Передача ссылок на массивы

Все функции (operations_sets, swap_pairs, merge_alternate) получают ссылки на массивы.

Это позволяет работать с исходными массивами без копирования данных.

#### menu_strings

```my %strings;``` Создаём пустой хеш %strings.

Ключи хеша — строки, значения — всегда 1 (заглушка).

Хеш используется для быстрого поиска и уникальности строк.

```while (1) { ... }``` Бесконечный цикл для повторного отображения меню после каждой операции.

Вывод меню:

print "\nМеню работы со строками ...";
print "1. Добавить строку\n";
...

```chomp(my $c = <STDIN>);``` Чтение ввода пользователя с удалением символа новой строки.

```$strings{$s} = 1;``` Добавляем строку в хеш как ключ.

Использование хеша автоматически гарантирует уникальность.


Сначала проверяем, есть ли ключ в хеше с помощью ```exists```

Если есть — удаляем ```delete $strings{$s}.```

Если нет — выводим сообщение "Такой строки нет.".

```my @out = sort keys %strings;``` Получаем ключи хеша (keys %strings) → все уникальные строки.

sort сортирует их лексикографически (по алфавиту).

```join("\n", @out)``` объединяет все элементы массива через символ новой строки, чтобы получился красивый список по одному элементу на строку.
```last;``` прерывает цикл и возвращает управление.

2️⃣ Главное меню
```
while (1) {
    print "\nГлавное меню:\n1. Работа с массивами\n2. Работа со строками (хеш)\n3. Выход\nВыберите: ";
    chomp(my $ch = <STDIN>);
    ...
}
```

Создаём бесконечный цикл, чтобы меню повторялось после выполнения операций.

Выход (last)

```chomp``` удаляет символ новой строки из ввода пользователя.

3️⃣ Обработка выбора пользователя

```$ch eq '1' ```→ вызываем функцию menu_arrays с ссылками на массивы.

```$ch eq '2'``` → вызываем функцию menu_strings.


```$ch eq '3'``` → оператор last завершает цикл и завершает программу.

4️⃣ Передача ссылок на массивы
```menu_arrays(\@array1, \@array2);```
Передаём ссылки на массивы, а не сами массивы.

Позволяет функциям изменять содержимое массивов напрямую без лишнего копирования.
